# Premium UI Rules

## Regras essenciais

1. A primeira dobra deve comunicar valor em menos de cinco segundos.
2. O layout deve parecer desenhado, nao apenas montado.
3. Cada bloco deve ter uma funcao comercial.
4. Espaco em branco e parte do design, nao vazio.
5. A tipografia deve carregar a sensacao premium.
6. CTAs devem ser claros, especificos e posicionados em momentos de decisao.

## Hero premium

Um hero premium deve conter:

- Mensagem principal concreta.
- Subtexto com resultado esperado.
- CTA primario.
- CTA secundario opcional.
- Prova rapida ou indicador de confianca.
- Elemento visual relevante ao servico, nao decoracao generica.

## Cards premium

Cards devem ser usados para comparar, agrupar ou destacar informacao.

Regras:

- Borda subtil.
- Sombra minima ou inexistente.
- Padding confortavel.
- Titulos curtos.
- Conteudo escaneavel.
- Um unico objetivo por card.

## CTAs

Usar verbos orientados a acao:

- Pedir diagnostico.
- Ver pacotes.
- Agendar chamada.
- Solicitar proposta.

Evitar:

- Saber mais em excesso.
- Clique aqui.
- Comecar agora quando o contexto ainda nao justifica urgencia.

## Densidade visual

Premium nao significa vazio. A pagina deve ser respiravel, mas com conteudo suficiente para sustentar decisao.

## Prova

Sempre que possivel, incluir:

- Resultados.
- Processos.
- Antes e depois.
- Stack de ferramentas.
- Exemplos de entregaveis.
- Testemunhos ou setores atendidos.

## Regra final

Se um elemento existe apenas para decorar e nao aumenta percecao, clareza ou conversao, deve ser removido.
